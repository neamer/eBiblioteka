Uvod
Primarni razlozi kreiranja ove aplikacije su seminarski radovi za predmete Razvoj softvera 1 i Interakcija čovjek-računar. Aplikacija nije responsivna zbog vremenskog ograničenja. Preporučene su veličine monitora 1366 x 768 i iznad.

Članovi tima su : Amer Nezirić i Amar Nezirić.

Informacije o aplikaciji su podijeljene u sljedeće dijelove :
  • koncept - ideja aplikacije, tipovi korisnika i osnovne funcije
  • Dizajn - Proces kreiranja i analiza dizajna i korištenih principa i uzorka
  • Tehnologije - Popis korištenih alata i tehnologija
  • Proces razvoja - Način na koji smo organizirali proces razvoja
  • Funkcionalnosti - Prikaz svih funkcionalnosti koje aplikacija pokriva
  • Raspodjela posla - Tabela koja prikazuje koji su udio imali članovi tima pri izradi specifičnih dijelova aplikacije
Koncept
eBiblioteka razlikuje dvije vrste korisnika, od kojih zavisi svrha korištenja aplikacije.

Bibliotekama se pruža automatizovan način upravljanja poslovanjem i uz to znatno olakšava posao bibliotekara. Bibliotekari mogu upravljati profilom biblioteke, autorima, knjigama i korisnicima.

Krajnjim korisnicima se pruža mogućnost korištenja usluga biblioteka bez potrebe da ih fizički posjete. Korisnici mogu pretražiti biblioteke, učlaniti se i iznajmiti knjigu online.
Dizajn
U sklopu izrade seminarskog rada iz predmeta Interakcija čovjek-računar, definisali smo opis i zahtjeve korisnika i na osnovu toga smo kreirali popis potrebnih funkcionalnosti za aplikaciju. Nakon toga je slijedio proces izrade wireframe-a koji je služio kao orijentir za organizaciju sadržaja. Sljedeći korak je bio detaljniji dizajn interakcije u slučajima gdje je to bilo potrebno. Dalje se na osnovu svega izradio high-fidelity dizajn za kompletnu aplikaciju. Također smo bili dužni uraditi detaljnu analizu dizajna koju možete pogledati na sljedećem linku : eBiblioteka_HCI_ZavrsniIzvjestaj
Tehnologije
Većinu tehnologija smo morali koristiti zbog zadovoljavanja kriterija za predmet Razvoj softvera.

Backend
  • Microsoft Visual Studio 2022 razvojno okruženje
  • .NET 5.0
  • Microsoft Entity Framework Core 5.0

Baza podataka
  • Microsoft SQL Server 2019
  • Microsoft SQL Server Management Studio

Frontend
  • Visual Studio Code
  • Angular 13
  • Typescript

Proces razvoja
Koristili smo SCRUM framework uz Azure DevOps za organizaciju procesa razvoja, kao što je predviđeno predmetom Razvoj Softvera 1. Aplikaciju smo uspjeli realizirati izvršavanjem 88 task-ova, koji su organizirani u 35 user-storya, u sklopu 10 sprintova.
